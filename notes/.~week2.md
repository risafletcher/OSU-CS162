```c++
#include <iostream>
#include <string>

#include <fstream>

int main(int args, char **args)
{
    std::ofstream ofs; 		// output file stream
    std::ofstream ofs("file.txt", std::ios::app);	// append flags can be passed here too
    std::ifstream ifs("file.txt"); // reading the file (input file stream)
    ofs.open("file.txt");	// open the file
    ofs.open("file.txt", std::ios::app);	// modifier flags: the append flag
    										// this prevent truncating 
    
    ofs << "Hello file!" << std::endl;
    
    std::string filename = "file.txt";
    std::ofstream ofs(filename.c_str(), std::ios::app);	// c_string for C++ 98
    
    std::vector<std::vector<int> > matrix;
    int v1, v2, v3, v4;
    // read from file and push to the matrix
    while(ifs >> v1 >> v2 >> v3 >> v4)
    {
        std::vector<int> v;
        v.push_back(v1);
        v.push_back(v2);
        v.push_back(v3);
        v.push_back(v4);
        matrix.push_back(v);
    }
    
    // print matrix
    for (int i = 0; i < 4; i++)
    {
        for (int j = 0; j < 4; j++)
            std::cout << matrix[i][j] << " ";
        std::cout << std::endl;
    }
    
    ifs.close();	// closes the input file stream
    
    return 0;
}
```

- Strings cannot be copied
- For strings, you must use `variableName.c_string()` to convert it to a C-string. C++ 11 can handle strings but because C++ 98 was created before this, C++ 98 compilers only know C-strings.
- `char c = ifs.get();`  will read the file one character at a time.
- `ofs.put(c)` will put the character to the output stream.

- flags:
  - `cout.setf(std::ios::FLAG)` turns flags on
  - `cout.unsetf(std::ios::FLAG)` turns flags off
- string streams:
  - `std::stringstream VARNAME` is a stream-like interface for working with strings.
  - once you’ve emptied a string stream, you don’t want to reuse it
- `getLine(fileStream, input);` will read an entire line of input until the new line. It reads the new line then discards it. The new line is not included in the input string. But it is taken off the input stream.
  - If the line is empty, you get an empty string.
  - if you have read everything on the line with `cin` and then call `getline` , you will get an empty string because there is nothing left on that line to give you.
  - Unlike regular `ifs`, `getline` will give you the next line, regardless of whether it’s empty or not.
  - Don’t mix `ifs` and `getline` in the same file due to this confusion.
