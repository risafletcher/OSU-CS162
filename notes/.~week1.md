### C Libraries and Compilation

#### C Libraries

When using C libraries, prefix the included file with “c”. Example: `#include <cmath>`. In C, this would be `#include <math.h>`.



#### Separate Compilation

The preprocessor runs, then the parser and lexer. The compiler translates the code into assembly, which is then turned into object code by the assembler. The linker then takes all of the object files and turns it into an executable.



#### Header Files

Headers are the interface of a series of related functions. Interface = prototypes.

The **include guard** prevents the redefinition of symbols. You can add an include guard by adding the preprocessor `#ifndef FILE_NAME_H`, then define with `#define FILE_NAME_H`. End your file with `#endif`.

An alternative to this is the `#pragma`, which is a preprocessor that tells the compiler to only include the header once. It does not work in all compilers on all platforms.



### Makefiles

#### Basic Terminology

`make` is a conditional build tool. If the target that is trying to build is older than any of the dependencies, it rebuilds.

Create a file called `makefile`.

All indentation is a `tab`.

Comments are prefixed with `#`.

```makefile
# Original:
lecture1: lecture1.cpp functions.hpp functions.cpp
	g++ -std=c++0x lecture1.cpp functions.cpp -o lecture1

# CC is the name of the C compiler.
# CXX is the name of the C++ compiler.
# CFLAGS & CXXFLAGS are the flags necessary to compile the 
# code but not necessarily to link it.
# LDFLAGS are flags for libriaries that need to be specified when 
# linking.

CC = gcc
CXX = g++
CFLAGS = -std=99
CXXFLAGS = -std=c++0x
# these are on separate lines so we can easily turn these on and off
# strictly enforce enforce standard and throw necessary errors
CXXFLAGS += -pedantic-errors
# turn on all warnings
CXXFLAGS += -Wall
# for debugging
CXXFLAGS += -g
# for optimization but should not be used when debugging
# CXXFLAGS += -O3

LDFLAGS = -lboost_date_time

OBJS = functions.o lecture1.o
SRCS = functions.cpp lecture1.cpp
HEADERS = functions.h

#target: dependencies
#	rule to build

# With variables:
lecture1: ${OBJS} ${HEADERS}
	${CXX} ${CXXFLAGS} ${OBJS} -o lecture1


# $(@:.o=.cpp) is an implicit rule
# the source file will end in .cpp and the 
# target is going to end in .o
# it will then build all the .o files from the .cpp source files
# ${OBJS} is an implicit target
${OBJS}: ${SRCS}
	${CXX} ${CSSFLAGS} -c $(@:.o=.cpp)
	
```



Use `make` to run it. Or, you can add a specific target to be built.
